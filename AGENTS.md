## gstack workflow

This repo uses gstack with Codex.

For complex implementation work, use $gstack-autoplan before editing.
For production-sensitive changes, use $gstack-careful and $gstack-guard.
For code review, use $gstack-review.
For QA/browser testing, use $gstack-qa.
For debugging, use $gstack-investigate.
For security/risk review, use $gstack-cso.
For release/shipping checks, use $gstack-ship.

Always report changed files, commands run, validation results, and remaining risks.
